# これまでの経緯(実施済み)
1. Next.js + TypeScript + Prisma(SQLite)+ NextAuth.js + Google Gemini API という
   構成で、数学の問題画像と手書き解答画像の2枚をアップロードすると、生成AI
   (マルチモーダルLLM)が解答を1ステップずつ検証し、最初に誤りが見つかった箇所を
   特定・添削するWebアプリケーション「MathCheck」を実装した。
   - AIは最終的な答えを直接提示せず、誤りステップの指摘とヒントのみを提示する設計。
   - 解答フローチャート(ステップごとの正誤)を画面に可視化する機能がある。
   - ログイン機能(Google/GitHub)、ユーザーごとの学習履歴のDB保存を実装済み。
2. 「ユーザー行動ブループリント」を作成し、フロントエンド/バックエンド/ユーザーの
   役割分担、行動選択の分岐(スキャン失敗時の再撮影、誤りがあった場合に続きを解くか
   終了するか等)を図示した。特に「解答フローチャートの生成→学習終了」の間の体験
   (つまずき箇所の可視化→ヒント/別解の選択→次の一歩のみ提示→再挑戦のループ)を
   このアプリの価値の核心として詳細設計した。

   概要: スマートフォンで撮影した数学の問題画像と手書き解答画像をもとに、生成AI
   (マルチモーダルLLM)が解答を1ステップずつ検証し、誤りの箇所を特定して添削する
   Webアプリケーションを設計・実装した。フロントエンド/バックエンド/DBの構成、

# 今、広げたいと考えている研究の方向性(キーワード)
現状は「数学の途中式添削」という具体アプリだが、これを以下のようなより広い研究テーマの
「一事例」として位置づけられないか検討している。

## ユーザーが抱える困りごと(問題側のキーワード)
- 解き方がわからない
- 違うやり方で解いたら正しいのかどうかわからない
- 次の一歩がわからない
- どこで間違えたのかわからない
- 解き終えたけど、正しいやり方かどうかわからない

## AIによる解決の方向性(解決側のキーワード)
- 生成AIが解き方を段階的に示してくれる
- 図示してくれる
- 次の一歩だけを示してくれる
- 「わからない」を「わかっていること」と「わからないこと」に分けてくれる

## 研究としての再定義候補(広げた場合の研究テーマ案)
- 「わからない」を「わかっていること」と「わからないこと」に分解し、言語化・形式化する
  AIアプリケーションの開発
- 「わからない」を見える化するAIの開発
- 「数学の問題」はその典型例の一つとして扱うと、研究としての広がりが出る
- 「わからない」「解けるけど理由が説明できない」を形式知に変換する生成AIの提案
- 暗黙知を形式知に変換する支援AIの開発
